# Style Guide: HowTo

## Purpose & Reader Intent

The reader wants to implement something specific — set up a chatbot, configure an integration, optimise a workflow. They want clear steps with enough context to understand why each step matters.

## Word Range

**Target: 800–1,200 words**

Step-based content. The StepList or HeroSteps component provides the overview; prose sections give detail and context. Match depth to task complexity.

## Structure Rhythm

1. **Lead** (1-2 sentences) — what you will accomplish and why it matters
2. **Prerequisites** (bulleted list) — what you need before starting
3. **Steps** (1-2 paragraphs per step) — action first, reasoning second
4. **Troubleshooting** (problem/solution pairs) — symptom-first format
5. **FAQ** (min 3 items, 2-4 sentences each)

## Section-Specific Guidance

### Steps

**Density:** 1-2 paragraphs per step
**What belongs here:** The action first ("Configure your webhook endpoint"), then the reasoning ("This ensures responses are routed correctly"). Include code snippets or configuration examples where relevant.
**What to cut:** Steps that just restate the overview. Excessive warnings for straightforward actions.

## Pre-Submission Checklist

- [ ] Every step leads with the action, not the reasoning
- [ ] Code/config examples included where relevant
- [ ] Troubleshooting uses symptom-first format
- [ ] Word count is within 800-1,200
