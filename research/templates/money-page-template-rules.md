# Money Page Template Rules (Enforced)

Applies to: all new `best-*` and commercial affiliate pages.

## 1) SERP-Proven title formula (required)
- Pattern: `Best <product> with <use-case> (<Year>): <N> Top Picks Tested` OR close variant.
- Must include:
  - Primary keyword in first 65 characters
  - Current year
  - Proof token (`tested`, `reviewed`, `compared`)

## 2) Fixed section order (required)
1. Intro (max 120 words)
2. TL;DR winner box
3. Comparison table
4. Product reviews by use-case
5. Buying guide
6. FAQ (3–5 questions)

If any section is missing, page is not publish-ready.

## 3) Comparison table schema (required columns)
- Product
- Best for
- Grinder type (Burr/Blade)
- Brew format (Single-serve/Carafe/Espresso)
- Capacity
- Key settings (grind/strength/program)
- Cleanup effort (Low/Med/High)
- Price band ($ / $$ / $$$)

## 4) Per-product block schema (required)
Each product block must include:
- 1-sentence verdict (who it is for)
- Pros list (2–4 bullets)
- Cons list (1–3 bullets)
- Key specs mini-list
- Why it wins this slot (explicit differentiator)

## 5) FAQ coverage (required intent targets)
Include at least one FAQ for each:
- Burr vs blade decision
- Is paying more worth it?
- Cleaning/maintenance frequency
- Best option for single-serve convenience

## 6) Internal linking minimums
Every money page must link to at least 3 relevant existing guides/posts.

## 7) Snippet optimization requirements
- Meta title: 55–60 chars target
- Meta description: 145–160 chars with value + use-case
- Add one concise answer box near top (40–60 words) to match PAA style.

## 8) Trust + methodology block
Must include short “How we evaluate” section before or near first product block, including:
- brew quality
- grinder consistency
- usability
- cleaning burden
- value for money

## 9) Budget-tier winner coverage (required)
For any page targeting a price ceiling (e.g., under $200), include explicit winner picks for:
- Best under $100
- Best $100–$149
- Best $150–$200

Each tier must have one named pick and one-line rationale.

## 10) Espresso capability labeling (required)
Comparison table and each product block must include:
- `Espresso-capable:` Yes / Borderline / No
- If `Borderline`, explain constraint in <= 20 words (e.g., stepped adjustment limits).

## 11) Freshness + maintenance proof signals (required)
All grinder money pages must include:
- `Prices checked: <Month Year>` stamp above the comparison table
- Maintenance scoring column in the table (`Low / Medium / High` cleanup effort)
- One sentence on retention/static behavior in every product block
